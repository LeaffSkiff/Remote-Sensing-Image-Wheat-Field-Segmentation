# 快速开始 - MMSegmentation 小麦田分割

## 环境准备

确保已安装并激活 conda 环境：
```bash
conda activate mmseg_env
```

## 使用步骤

### 1. 准备数据集

运行以下命令将数据转换为 MMSegmentation 格式：

```bash
# 使用 conda 环境的 Python
C:\Users\28027\miniconda3\envs\mmseg_env\python.exe mmseg\tools\prepare_mmseg_data.py
```

这将在 `data/processed/` 目录下创建：
```
data/processed/
├── images/
│   ├── train/    # 训练图像 (80%)
│   ├── val/      # 验证图像 (20%)
│   └── test/     # 测试图像
└── annotations/
    ├── train/    # 训练标注
    └── val/      # 验证标注
```

### 2. 开始训练

**方法一：使用批处理脚本（推荐）**

双击运行 `scripts/train.bat`，然后按菜单选择操作。

**方法二：直接命令行**

```bash
# 训练 DeepLabV3+
C:\Users\28027\miniconda3\envs\mmseg_env\python.exe ^
  C:\Users\28027\miniconda3\envs\mmseg_env\Lib\site-packages\mmseg\tools\train.py ^
  mmseg\configs\wheat\wheat_deeplabv3_r50.py ^
  --work-dir mmseg\work_dirs\wheat_deeplabv3

# 训练 UNet
C:\Users\28027\miniconda3\envs\mmseg_env\python.exe ^
  C:\Users\28027\miniconda3\envs\mmseg_env\Lib\site-packages\mmseg\tools\train.py ^
  mmseg\configs\wheat\wheat_unet_r50.py ^
  --work-dir mmseg\work_dirs\wheat_unet
```

### 3. 评估模型

```bash
C:\Users\28027\miniconda3\envs\mmseg_env\python.exe ^
  C:\Users\28027\miniconda3\envs\mmseg_env\Lib\site-packages\mmseg\tools\test.py ^
  mmseg\configs\wheat\wheat_unet_r50.py ^
  mmseg\work_dirs\wheat_unet\iter_40000.pth ^
  --eval mIoU
```

### 4. 预测新图像

```bash
C:\Users\28027\miniconda3\envs\mmseg_env\python.exe ^
  mmseg\tools\predict_mmseg.py ^
  --config mmseg\configs\wheat\wheat_unet_r50.py ^
  --checkpoint mmseg\work_dirs\wheat_unet\best.pth ^
  --image to_predict\your_image.png ^
  --output predictions
```

## 配置文件说明

| 文件 | 说明 |
|------|------|
| `mmseg/configs/wheat/wheat_unet_r50.py` | UNet 模型配置 |
| `mmseg/configs/wheat/wheat_deeplabv3_r50.py` | DeepLabV3+ 模型配置 |

## 训练参数调整

编辑配置文件可以调整：

- `num_classes=2` - 类别数（背景 + 小麦田）
- `batch_size=4` - 批次大小（根据显存调整）
- `max_iters=40000` - 训练迭代次数
- `lr=0.01` - 学习率

## 输出目录

训练结果保存在 `work_dirs/`:
- `best.pth` - 最佳模型
- `latest.pth` - 最后一次迭代
- `log.txt` - 训练日志

## 常见问题

**Q: 显存不足？**
减小配置文件的 `batch_size` 或 `crop_size`

**Q: 训练太慢？**
减少 `max_iters`，或使用更小的 backbone

**Q: 找不到 mmseg 模块？**
确保使用 conda 环境的 python.exe 运行
